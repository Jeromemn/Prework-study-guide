HTML

  • HTML gives the content structure.
  • The head contains metadata, which helps the browser render the page correctly.
  • Head also contains: Style, Base, Link, Script.
  • The body element represents the visual content seen by user.
  • Parent element: An element that holds other elements inside.
  • Child element: Elements that are nested inside other elements.
  • Metadata contains information about accessibility, search engines and performance.
  • Metadata tags: Title, Style, Base, Link, Meta.
  • Semantic HTML element: The name of the element describes its content and the role it plays on webpage. Ex: Header, Footer, Main, Section.
  • Non Semantic element: Name of element doesnt describe its content, ex: "Div" or "Span".
  • HTML elements are essentially in a box, consisting of margins outside the element, the element borders, padding and content.

CSS

  • Three ways to add style; inline CSS, Internal CSS and external CSS.
  • Use * to apply a rule to all elements on a visible page.
  • Can Combine multiple selectore using a ",".
  • Examples of selectors: "img", "*", "header", "footer", "class".
  • Properties: "display", "height", "width", "margin", "box-shadow", "border".
  • Assigning colors: Can be done in semantic referiences "blue" and can refer to hexadecimal code Hex#.
  • Selectors: Img, *, Header, Footer, class.

GIT

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git status: Checks what branch you are currently working.
  • GIT checkout -b "branch name": Creates a new branch and switches to it.
  • Git add -A: Adds modifications in the current working branch to the staging branch, the "-A" indicates we want to add all changes. Informs Git changes will be added in next commit.
  • Git add "file name": Add a file as it looks now to your next commit.
  • Git Reset "file name": Unstage a file while retaining the changes in a working directory.
  • Git dif: Difference of what is changed but not staged. Git diff --staged:Difference of what is staged but not yet commited.
  • Git commit -m "description": Commit your staged content as a new commit snapshot.
  • Git clone "Paste copied SSH value": Clones GitHub repositiory.
  • Git pull origin main: Pull main branch from Github repository.
  • Git push origin "file name": Push local changes to remote Github branch.

JAVASCRIPT

  • Allows Users to interact with websites.
  • JavaScript is read top to bottom.
  • Primitive values: Boolean, Null, Undefined, Number type, Bigint type, String type, Symbol type.
  • Objects: A value in memory referenced by a identifier; Data property and Accessor Property.
  • Indexed collections: Arrays and Typed Arrays.
  • Keyed Collections: Maps, Sets, Weakmaps and Weaksets.
  • Variable: A named container that allows us to store data in our code.
  • Methods: Log set of instructions that can be executed by a computer.
  • Console Test environment developers use to check their code such as Chrome Dev.
  • Control Flow: The order in which a computer executes code in a script.
  • Condition Statements: Control order in which code is executed or executed at all. Will only execute if the statement meets a condition or is true.
  • Loops: Control order in which code is code is executed. Uses predictable pattern to perfrom a task on all items in a array by allowing a single block of code to repeat.
  • Array: Single Variable that is used to hold a group of data.
  • Index: Unique number that identifies the location of individual data. First item in an Array has a index of 0 and increase by 1 each time.
